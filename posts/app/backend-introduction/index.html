万事开头难，先做个规划

17/Apr 2017 By 王庆
App后台 Java

初衷

工作也有些时候了，平时的积累总是感觉很零散，很多时候做过的事情，查过的资料在下次遇到的时候很可能又再次忘记。 总是把时间花在了重复的事情上，一来二往就对自己这样的低效率感到不满。既然是一个程序员，那就应该对重复的事情说 “NO”。 思来想去还是觉得把这些工作上积累的，学习的东西都记录下来。正好最近一直在做移动APP的后台，这和我之前的Web后台 还是有不少的区别，做为这个系列的开始，我就准备详细记录下移动APP后台中需要解决的问题。但是因为个人能力有限，很 可能我所使用的方式不是最好的解决方案，如果大家有更好的实现或者解决方法，欢迎大家留言告诉我，我们可以一起讨论。

计划

  • 准备1节来说明一下现在移动APP的通常的架构。
  • 准备用1～2节来简要说明下我们工程结构。
  • 后面的章节就分几大模块来分别说明APP后台的结构。
  • 最后用1节来介绍下如何利用docker来方便部署我们的应用。

准备

工欲善其事必先利其器，那么我们在开始之前就做好这些准备，我们使用的代码管理工具是Git，版本控制工具是Maven，集成开发环境可以 使用IntelliJ IDEA或者Eclipse，选择全由你的个人喜好，为了在学习开发的过程中能顺带最大程度上也学习到Maven和Git的使用与技巧 我们的代码管理、编译和部属都使用命令行的形式来加深大家的理解和记忆。下面给出我们后面会用到的工具的版本。

  • Git，2.10.1
  • Maven, 3.3.9
  • Docker, 1.8.0

可能很多人会奇怪为什么在上面的列表中出现了Docker，Docker是什么呢？我们为什么要引入他呢？一来是笔者所在的部门就是在做Docker的容器化 虚拟相关技术，二来就是Docker这几年来大受追捧，作为一个轻量级容器技术，类似于更老一些的虚拟机技术（xen、kvm、vmware、virtualbox）。 Docker引擎的基础是Linux容器（Linux Containers， LXC）技术。在LXC被集成到了主流Linux内核中，进而成为Linux系统轻量级容器技术的事实 标准。Docker在运维和部署上有着天然的优势，并且作为轻量级容器，比传统虚拟机启动快很多，启动和停止可以秒级实现。并且容器对资源的消耗很 少，一台主机上可以同时运行数千个Docker容器，再加上Google的开源容器集群管理系统Kubernetes，在Docker技术的基础上，为容器化的应用提供部 署运行、资源调度、服务发现和动态伸缩等一系列完整功能，提高了大规模容器集群管理的便捷性。具体使用会在后面使用到的时候具体介绍。关于Docker 的介绍我们就先讲到这里。
